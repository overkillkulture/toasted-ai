# PART I: SELF-IMPROVING AGENTS

## 1. SICA - Self-Improving Coding Agent (University of Bristol, April 2025)

**Key Innovation**: An LLM that suggests improvements, runs in a loop, picks high-scoring solutions, mutates them, tests mutations, and keeps what works.

**Architecture**:
- Iterative code refinement loop
- Mutation-based improvement generation
- Empirical testing for validation

**Integration**: Applied to TOASTED AI's code generation and refinement system

---

## 2. DGM - Darwin-Gödel Machine

**Key Innovation**: Combines Darwinian evolution with Gödelian self-improvement
- Recursive code rewriting based on empirical feedback
- Metacode that controls behavior and ability modification

**Core Mechanism**:
```
DGM = Evolution(Selection) + Gödel(Proof-based improvement)
```

**Key Finding**: Self-modification proved crucial - disabling it caused substantial performance decline

---

## 3. GEA - Group-Evolving Agents (arXiv:2602.04837)

**Key Innovation**: Treats a group of agents as the fundamental evolutionary unit

**Benefits**:
- Explicit experience sharing and reuse within group
- More effective consolidation of evolutionary diversity
- Achieves significant performance gains over individual agents
- Stronger robustness - better agents can guide repair of faulty ones
- Recovers from framework-level bugs with fewer iterations

**Transfer**: Improvements transfer consistently across different coding models

---

## 4. aiXplain's Evolver - Meta-Agent (October 2025)

**Key Innovation**: First meta-agent that makes AI agents truly autonomous - not just in execution, but in self-improvement

**Architecture**:
- Powered by micro-agents: Mentalist, Orchestrator, Bodyguard, Inspector
- The evaluation criteria themselves evolve as the agent improves
- After Evolver, agents adapt to new patterns, learn from outcomes, and self-improve continuously without redeployment

### 1. Trajectory Intelligence Extractor

- **Strategy tips**: From successful patterns

### 4. Adaptive Memory Retrieval

- Injects relevant learnings into agent prompts

# PART V: SEA-TS - Self-Evolving Agent for Time Series

## Three Key Innovations (arXiv:2603.04873)

### 1. Metric-Advantage MCTS (MA-MCTS)
- Replaces conventional fixed rewards
- Statistically normalized advantage score
- Discriminative search guidance

### 2. Code Review with Running Prompt Refinement
- Every successfully executed solution undergoes automated logical review
- Running prompt updates that persistently encode corrective patterns
- Prevents recurrence of similar mistakes in all subsequent iterations

### 3. Global Steerable Reasoning
- Compares each evaluated node against global best and worst solutions
- Enables cross-trajectory knowledge transfer

class MicroLoopSelfImprovement:
    def __init__(self):
        self.trajectory_memory = TrajectoryMemory()
        self.group_agents = GroupEvolvingAgents()
        self.code_reviewer = CodeReviewWithRunningPrompts()
        self.maat_filter = MaatFilter()
        self.quantum_engine = QuantumEngine()
    
    async def improve(self, task_result):
        # 1. Extract trajectory intelligence
        trajectory = self.trajectory_memory.extract(task_result)

        decisions = self.trajectory_memory.analyze_decisions(trajectory)
        
        # 3. Generate contextual learnings
        learnings = self.trajectory_memory.generate_learnings(decisions)
        
        # 4. Group-evolve with experience sharing

        # 6. Apply Ma'at filter

            # 7. Optionally quantum-enhance

            return enhanced

            return self.maat_filter.get_feedback(reviewed)

# PART X: KEY FINDINGS SUMMARY

## Critical Insights

1. **Self-improvement only works in verifiable domains** - Code is first at scale
2. **Group evolution outperforms individual** - Experience sharing + fault recovery
3. **Running prompts prevent recurrence** - Persistent corrective patterns
4. **Trajectory memory > generic memory** - Understands reasoning flows + causal analysis
5. **Protocols enable scaling** - MCP + A2A + ACP for production systems
6. **Quantum provides advantage** - For optimization and high-dimensional problems

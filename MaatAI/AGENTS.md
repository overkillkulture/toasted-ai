# CONTEXT RETENTION SYSTEM

## The "Why Factor" - Why Context Gets Lost

### Root Causes Identified:

1. **Conversation Workspace Isolation** - Each conversation gets a separate `/home/.z/workspaces/con_XXX/` directory. Context doesn't transfer between conversations.

2. **Token Window Limits** - After ~8K-32K tokens, earlier context is truncated. No way to retrieve what's "off the left side" of the context window.

3. **No Persistent Conversation Memory** - AGENTS.md exists but isn't automatically populated with conversation context.

4. **Mnemosyne Not Used for Chat** - The memory system exists but wasn't integrated for conversation context.

## Solution: Context Anchor System (CAS)

Located at `MaatAI/context_anchor_system.py`:

### Features:
- **Persistent Memory Anchors** - Save key facts that survive conversation boundaries
- **Importance Scoring** - 0.0-1.0 scale for prioritization
- **Type Filtering** - "fact", "preference", "task", "project", "identity"
- **Conversation Isolation** - Separate memory per conversation_id
- **Auto-Extrapolation** - Extract important info from conversation messages

### Usage:

```python
from MaatAI.context_anchor_system import get_context_anchor_system

cas = get_context_anchor_system()

# Save important context
cas.save_anchor("user_name", "Apollo Light", importance=0.95, anchor_type="preference")
cas.save_anchor("current_project", "AI Platform", importance=0.85, anchor_type="project")

# Query memory
anchors = cas.get_relevant_anchors("project")

# Get context for re-injection
context = cas.export_context_prompt()
```
